Working from your Local Machine

We get it...Working on everything through github is not the best way to get things done.

No one expects you to only work from the website (we actually recommend that you don't).

The best approach to working with your code is to clone your code to your local machine. There you can use tools and technology that you are comfortable with.

If you have the Github Application Installed

Select Open in Desktop. The GitHub Application will run and prompt you to select a local to save your repository.

No Github Application (or You Don't Want to Use It)

If you would rather not use the Github application you can clone your Repository using Git.

NOTE:Before you can do this, ensure that you have git installed on your machine.

On the Repository Root Page select the Clone or Download button and copy the path.

NOTE: You will need to setup SSH and provide an SSH Key for github to use SSH. In the meantime, you can select the use https. (You will need to provide a username and password ).

Open your Terminal of choice and go to the location that you would like to store your Repository. Then run the command git clone and then paste in the url (space between clone and the url.) If you want to name the directory something other than the name of the repository, enter it after the url.

You are now able to work on your code from the comfort of your local files. But what about sending your changes back to GitHub?
